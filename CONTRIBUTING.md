# Contributing to FlowForge

Thank you for your interest in contributing to FlowForge.

Contributions of all sizes are welcome, whether it's fixing bugs, improving documentation, enhancing the UI, or implementing new features.

---

## Getting Started

Clone the repository.

```bash
git clone https://github.com/yourusername/flowforge.git
```

Install dependencies.

```bash
npm install
```

Start the development server.

```bash
npm run dev
```

---

## Development Guidelines

Please follow these principles when contributing.

### Code Style

- Use meaningful variable names.
- Prefer reusable components.
- Keep components focused on a single responsibility.
- Avoid unnecessary duplication.
- Write clean and readable code.

---

### Folder Structure

Follow the existing project organization.

```
components/
pages/
store/
services/
layouts/
routes/
```

Avoid creating unnecessary folders or deeply nested structures.

---

### Commits

Use descriptive commit messages.

Examples:

```
feat: add task editing

fix: resolve notification bug

refactor: simplify sidebar logic

docs: update README
```

---

### Pull Requests

Before opening a pull request:

- Ensure the project builds successfully.
- Test your changes.
- Keep pull requests focused on a single feature or fix.
- Update documentation when necessary.

---

## Reporting Issues

When reporting a bug, please include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Browser and operating system

---

## Suggestions

Feature requests and improvement ideas are always appreciated.

Thank you for helping improve FlowForge.
